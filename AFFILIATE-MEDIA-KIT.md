## Post-approval checklist

For each program that approves:

1. Copy the tracking URL from the program dashboard
2. If the program supports sub-IDs: append `?source={source}` (or
   `?clickref={campaign}` for PartnerStack) to get per-page attribution
3. Add `ALTAI_AFFILIATE_<SLUG>` on Vercel (see `ENV-AFFILIATES.md` for
   the slug → env-var table)
4. If the program forbids UTM parameters (rare), also add
   `ALTAI_AFFILIATE_<SLUG>_NO_UTM=1`
5. Trigger a Vercel redeploy
6. Verify by clicking the "Try X →" button on a tool page → confirm
   the URL in the browser matches the tracking URL, and your
   attribution click fires in the program dashboard within ~5 min

## If a program rejects you

Common reasons + fixes:

- **"Site not live yet / insufficient traffic"** — get the custom domain
  live (LAUNCH-CHECKLIST §3) and wait 2-4 weeks for initial indexing.
  Re-apply with updated traffic numbers.
- **"Content thin / programmatic"** — add 4-6 blog posts (`data/tools.json`
  → `blog` array → rebuild) covering hand-written editorial on 2-3 of
  the top tools. Most programs soften on this after one round of content.
- **"Competes with us"** — if a network rep flags a specific comparison
  page as unfavorable to their advertiser, offer to add a balancing
  paragraph. Do NOT compromise the ranking if it reflects actual tool
  capability.
- **"Niche fit unclear"** — link directly to the `/tools/<vendor>-alternatives.html`
  page in your re-application; the dedicated page makes the fit obvious.

Rejection on one program does not affect the others. Re-apply after
90 days if the first pass rejected on "insufficient traffic."
